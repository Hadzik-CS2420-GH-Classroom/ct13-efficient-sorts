👨‍💻 Code-Together 13: O(n log n) Sorting Algorithms

🎯 Overview

An in-class code-together activity introducing efficient sorting algorithms — merge sort, quick sort, and heap sort. These divide-and-conquer algorithms achieve O(n log n) time by splitting the problem in half at each level, doing O(n) work per level. Students walk through the instructor-provided EfficientSorts.cpp implementation and complete TODOs in main.cpp to exercise each algorithm. The scenario: sorting exam scores.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Implement merge sort: split the array, recursively sort each half, merge two sorted halves with the two-pointer technique
  • Implement quick sort: choose a pivot (median-of-three), partition elements around it, recursively sort each side
  • Implement heap sort: build a max-heap in-place, repeatedly extract the max to produce sorted output
  • Explain why all three are O(n log n) — log n levels of recursion/extraction, O(n) work per level
  • Compare space tradeoffs: merge sort uses O(n) extra space, quick sort uses O(log n) stack, heap sort uses O(1)
  • Explain stability: merge sort is stable; quick sort and heap sort are not

📚 Teaching Order

Work through the files in order:

1. EfficientSorts.h — Function declarations

Walk through the header. Compare to CT12’s quadratic sorts: same interface (vector<int>&), dramatically different internals.

Section What You’ll Learn

quick_sort() O(n log n) avg, O(n²) worst, O(log n) space, not stable. Partition strategy.
heap_sort() O(n log n) always, O(1) space, not stable. Max-heap + extract strategy.

2. EfficientSorts.cpp — Sort implementations (0 TODOs)

All code is given — students follow along as the instructor walks through each method.

Section What You’ll Learn
1. merge() helper Copy halves into temp vectors, merge with two pointers, <= for stability
2. merge_sort_recursive() Base case (left >= right), find mid, recurse left, recurse right, merge
3. partition() helper Median-of-three pivot selection, scan with i/j pointers, place pivot
4. quick_sort_recursive() Base case (low >= high), partition, recurse left side, recurse right side
5. heapify_down() helper Compare with children (2i+1, 2i+2), swap with largest, recurse
6. heap_sort() Phase 1: build max-heap (bottom-up). Phase 2: extract max repeatedly.

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct13-efficient-sorts.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ! DISCUSSION: comments in each file — these explain the “why” behind each concept. The // ? SEE DIAGRAM: comments point you to visual aids in the images/ folder.
